## 一、游戏概述

**静夜标记 (Silent Mark)** 是一款基于狼人杀框架的在线多人社交推理桌游。

核心特点：移除自由发言环节，玩家通过放置"标记"来声明身份、评价他人、传递信息，并通过公开投票放逐狼人。

### 1.1 核心设计目标

- **零语言门槛**：完全无需语音或文字交流
- **易学难精**：核心规则 5 分钟可理解，策略深度足够
- **节奏明快**：固定标记操作替代冗长讨论，单局 10-15 分钟
- **强社交推理**：通过公开的标记网络进行心理博弈和逻辑对抗

### 1.2 支持人数

游戏支持**任意人数**，由房主在创建房间时自定义各职业的数量。

#### 推荐配置（预设模板）

| 模板名 | 人数 | 阵营配置 |
|--------|------|---------|
| 6人标准局 | 6人 | 2狼人 + 预言家 + 女巫 + 猎人 + 1平民 |
| 9人标准局 | 9人 | 3狼人 + 预言家 + 女巫 + 猎人 + 守卫 + 2平民 |

房主也可以完全自定义：选择每种角色的人数，系统校验合法性后开始游戏。

## 二、角色设计

### 2.1 角色一览

| 角色 | 阵营 | 夜间技能 | 专属标记理由 |
|------|------|---------|-------------|
| 预言家 | 好人 | 查验一人阵营（好人/狼人） | 【查验结论】 |
| 女巫 | 好人 | 持有解药×1、毒药×1 | 【用药结果】 |
| 猎人 | 好人 | 出局时可开枪带走一人 | 无 |
| 守卫 | 好人 | 守护一人（不可连续守同一人） | 无 |
| 平民 | 好人 | 无 | 无 |
| 狼人 | 狼人 | 夜晚共同选择袭击一人，同阵营互认 | 无（需伪装好人身份） |

### 2.2 角色详细规则

#### 预言家
- 每晚选择一名玩家查验，得知其阵营（好人/狼人）
- 查验结果仅自己可见
- 可在评价标记中使用专属理由【查验结论】

#### 女巫
- 持有解药和毒药各一瓶，全局各限用一次
- 夜晚先得知当晚被狼人袭击的玩家
- 可选择：使用解药救人 / 使用毒药毒一人 / 不操作
- **首夜可以自救，非首夜不可自救**
- 同一晚不可同时使用解药和毒药
- 可在评价标记中使用专属理由【用药结果】

#### 猎人
- 被**放逐**出局时：可以选择开枪带走一名存活玩家
- 被**狼人刀死**时：可以选择开枪带走一名存活玩家
- 被**女巫毒死**时：**不能开枪**
- 开枪为可选操作，可以选择不开

#### 守卫
- 每晚选择守护一名玩家（可以守自己）
- **不可连续两晚守护同一人**
- 守护成功则该玩家免疫狼人袭击
- **同守同救规则**：若守卫守护的玩家同时被女巫用解药救，该玩家死亡（同归于尽）

#### 平民
- 无任何特殊技能
- 通过标记和投票参与游戏

#### 狼人
- 夜晚阶段所有狼人共同商议（系统内选择）袭击目标
- 狼人之间互相知道身份
- 白天标记阶段必须伪装成好人身份

### 2.3 角色出局方式汇总

| 出局方式 | 猎人能否开枪 | 身份是否公开 |
|----------|-------------|-------------|
| 被狼人刀死 | ✅ 能 | 不公开 |
| 被女巫毒死 | ❌ 不能 | 不公开 |
| 被放逐 | ✅ 能 | 不公开 |

**所有出局方式均不公开身份**，玩家身份仅在游戏结束时统一揭晓。

## 三、标记系统

标记是本游戏的核心机制，替代传统狼人杀的自由发言环节。

### 3.1 标记类型

| 标记类型 | 目标 | 每轮数量 | 必须使用 | 说明 |
|---------|------|---------|---------|------|
| 身份声明标记 | 自己 | 1 | 是 | 声明自己的身份 |
| 评价标记 | 其他存活玩家 | 2-4（取决于当局人数） | 是 | 评价他人身份 |

#### 评价标记数量规则

评价标记数量随当局**存活玩家总数**动态调整：

| 存活玩家数 | 评价标记数量 |
|-----------|-------------|
| 4-6 人 | 2 个 |
| 7-9 人 | 3 个 |
| 10 人及以上 | 4 个 |

### 3.2 标记身份选项

放置标记时可选择的身份标签**取决于当局参与的职业**：

#### 固定选项（始终可用）
- **神职** — 模糊选项，表示"我是神但不说是哪个"
- **好人** — 泛指好人阵营
- **狼人** — 评价他人为狼人（仅用于评价标记，身份声明不可选）

#### 动态选项（当局包含该职业时才出现）
- 预言家（当局有预言家时）
- 女巫（当局有女巫时）
- 猎人（当局有猎人时）
- 守卫（当局有守卫时）
- ...（后续扩展角色同理）

**示例**：若房主配置的当局没有守卫，则标记身份选项中不会出现"守卫"。

### 3.3 标记理由

每个标记必须附带一个理由。

#### A. 通用理由（所有玩家、所有标记类型均可用）

| 理由 | 说明 |
|------|------|
| 直觉判断 | 基于模糊的整体感觉 |
| 基于投票的分析 | 分析目标玩家的历史投票行为 |
| 基于标记的分析 | 分析目标玩家的历史标记行为 |
| 基于日志的推理 | 结合系统公布的夜间日志推导 |

#### B. 专属理由（声明对应身份后，仅在评价/归票标记中可用）

| 理由 | 可用条件 |
|------|---------|
| 【查验结论】 | 本轮身份声明为预言家 |
| 【用药结果】 | 本轮身份声明为女巫 |

**注意**：身份声明标记只能使用通用理由。

### 3.4 标记流程与公开规则

标记阶段类似传统狼人杀的"发言环节"，**按座位顺序依次进行，所有操作实时公开**。

#### 流程
1. 系统按座位顺序（或随机顺序）确定标记发言顺序
2. 轮到某玩家时，该玩家在限定时间内完成所有标记操作：
   - 放置 1 个身份声明标记（给自己）
   - 放置 2-4 个评价标记（给其他玩家）
3. 该玩家确认提交后，其所有标记**立即公开**，所有人可见
4. 轮到下一位玩家操作
5. 所有玩家完成后，标记阶段结束

#### 关键要点
- 后发言的玩家可以看到前面所有人的标记，这与传统狼人杀的发言顺序逻辑一致
- 发言顺序本身是策略要素之一
- 所有历史标记可随时查看，支持按轮次筛选

## 五、夜晚系统

### 5.1 夜晚行动顺序

```
守卫 → 狼人 → 女巫 → 预言家
```

1. **守卫**：选择守护一名玩家
2. **狼人**：所有狼人共同选择袭击目标
3. **女巫**：得知被袭击者后，选择是否用药
4. **预言家**：选择查验一名玩家

每个角色有固定操作时间（倒计时），超时视为不操作。

### 5.2 夜晚结算逻辑

```
1. 狼人选择袭击目标 A
2. 若守卫守护了 A → A 免疫袭击（但若女巫同时对 A 用解药 → 同归于尽，A 死亡）
3. 若守卫未守护 A → A 被袭击
   - 若女巫对 A 使用解药 → A 存活
   - 若女巫不救 → A 死亡
4. 女巫可额外选择对某人使用毒药 → 该人死亡
5. 预言家查验结果仅自己可见
```

### 5.3 夜晚公告

夜晚结束后，系统**仅公布死亡信息**，与传统狼人杀一致：

- 有人死亡：公布死亡玩家编号（不公布身份）
- 无人死亡：公布"平安夜"
- 若一晚多人死亡（如被刀 + 被毒），同时公布

不额外公布守护、查验等信息，这些由玩家通过标记系统自行传递。

### 5.4 随身物品系统

游戏开始时，每位玩家**随机获得一种随身物品**。每种物品只能记录**一个维度**的信息，玩家无法选择物品类型。

#### 核心规则

- 物品类型和内容：**存活时仅自己可见，死亡后连同类型一起向所有人公开**
- 物品类型**完全随机分配**，不控制同局内各类型的数量
- 物品只记录一个**数字**，代表该维度事件的累计总次数

#### 物品类型

| 物品 | 记录内容 | 显示 | 直觉理解 |
|------|---------|------|---------|
| **镜** | 被任何夜间行动造访的总次数 | 数字 | "多少次行动指向过我" |
| **痕** | 被狼人选为袭击目标的总次数 | 数字 | "被狼人盯上几次" |
| **待定** | 待定 | 数字 | 待定 |

**镜的触发条件**：被狼人选为袭击目标 / 被预言家查验 / 被守卫守护 / 被女巫用药（解药或毒药），每次 +1

**痕的触发条件**：被狼人选为袭击目标（无论是否存活），每次 +1

#### 信息不对等

同一个数字，不同角色能解读出不同的信息：

**示例**：玩家 A 死亡，物品公开为「镜：4」

- **预言家**：我查过 A 一次，所以除我之外还有 3 次其他行动指向 A
- **守卫**：我守过 A 两晚，所以除我之外还有 2 次其他行动
- **狼人**：我们刀了 A 两次，所以除我们之外还有 2 次其他行动（有神职在关注 A）
- **普通好人**：只知道 A 被关注了 4 次，无法拆分

**示例**：玩家 B 死亡，物品公开为「痕：2」

- **狼人**：没有新信息（自己刀的当然知道）
- **女巫**：B 被刀过 2 次，说明第一次被救了（可能是我救的，也可能是守卫守的）
- **普通好人**：B 被刀过 2 次却到现在才死，说明之前被保护过一次

#### 存活时的自我信息

物品给活着的玩家提供**被动获取的私有信息**：

- 拿到「镜」的平民发现数字从 0 变成 1 → "昨晚有人对我执行了行动"，但不知道是查验、守护还是被刀未遂
- 拿到「痕」的玩家发现数字 +1 但自己没死 → "我昨晚被狼人选中了但被救/被守了"
- 这些私有信息可通过标记系统间接传递

#### 设计要点

- 每种物品**只记录一个维度的累计数字**，不记录时间序列，不记录行动来源
- 物品类型随机分配，可能出现一局内大部分人拿同一种的极端情况——这是有意为之，增加戏剧性
- 不能设计任何**无论如何操作都会直接暴露持有者身份**的物品

> **⚠️ 待讨论（物品系统仍在探索阶段）**：
> - 第三种物品的设计方向尚未确定。曾讨论过"记录被保护次数"的方向，但与「痕」信息高度耦合（痕 - 被保护次数 ≈ 1），需要找到与镜、痕正交的信息维度
> - 物品命名（镜 / 痕 / 待定均为暂用名）
> - 是否需要在标记理由中增加"基于物品的推理"选项，打通物品系统与标记系统

## 六、白天流程

白天是玩家通过标记进行信息交换和推理的核心阶段，整体流程对标传统狼人杀的"白天发言→投票"。

### 6.1 完整白天流程

```
公布死讯 → 猎人开枪（如触发）→ 检查胜负 → 标记发言 → 放逐投票 → 猎人开枪（如触发）→ 检查胜负
```

### 6.2 公布死讯

- 显示昨夜死亡的玩家（**不公开身份**）
- 若无人死亡，显示"平安夜"
- 同时公布本夜的系统日志

### 6.3 标记发言阶段

标记发言是本游戏的核心环节，替代传统狼人杀的自由发言。

#### 流程
1. 系统确定本轮发言顺序（首轮随机，后续轮次可沿用或调整）
2. 按顺序轮到每位存活玩家时：
   - 该玩家在限定时间内完成标记操作：
     - **1 个身份声明标记**（给自己，选择身份 + 理由）
     - **2-4 个评价标记**（给其他存活玩家，选择身份 + 理由）
   - 玩家确认提交后，其标记**立即公开**，全体可见
3. 所有玩家完成后，标记发言阶段结束

#### 要点
- 后发言的玩家可以看到前面所有人的标记，据此调整自己的策略
- 发言顺序是影响博弈的重要因素
- 每位玩家有独立的操作倒计时，超时则系统自动提交已完成的标记（未完成的部分跳过）

### 6.4 放逐投票

- 标记发言结束后，进入投票阶段
- 所有存活玩家**同时投票**，选择放逐一人
- 不可弃票（必须投）
- 不可投自己
- 得票最高者被放逐（**不公开身份**）
- **平票则无人出局**
- 投票结果（谁投了谁）在投票结束后**公开**

### 6.5 遗言（房间可选项，默认关闭）

- 被放逐者可发表简短文字遗言
- 此功能为房间设置项，房主可开启/关闭
- 默认关闭以保持"零语言"纯粹性

## 七、胜负判定

### 7.1 好人阵营胜利

所有狼人被放逐或击杀。

### 7.2 狼人阵营胜利

达成**屠边**：
- 所有神职角色出局（预言家、女巫、猎人、守卫全部死亡），或
- 所有平民出局

### 7.3 判定时机

每次有玩家出局后立即检查胜负条件：
- 夜晚结算后
- 放逐后
- 猎人开枪后

## 八、房间系统

### 8.1 创建房间

- 房主选择游戏模式（6人局 / 9人局）
- 房主可配置：
  - 遗言开关（默认关闭）
  - 标记阶段时长（默认 60 秒）
  - 投票阶段时长（默认 30 秒）
  - 夜晚操作时长（默认 20 秒）
- 创建后生成房间号（6位数字）

### 8.2 加入房间

- 输入房间号加入
- 设置昵称（房间内唯一）
- 加入后进入等候大厅

### 8.3 等候大厅

- 显示当前房间内所有玩家
- 房主可踢人
- 房主点击"开始游戏"（需人数满足要求）
- 所有玩家可看到房间配置

### 8.4 掉线处理

- 玩家掉线后保留位置 60 秒，期间可重连
- 超时未重连：
  - 夜晚行动阶段：视为不操作
  - 标记阶段：系统自动随机完成标记
  - 投票阶段：系统自动随机投票
- 不做 AI 托管

## 九、完整单局流程

```
1. 房主创建房间，玩家加入
2. 房主点击开始
3. 系统分配身份（仅本人可见）
4. 狼人互相确认同伴

=== 第一天 ===

5. 第一个夜晚
   - 守卫选择守护目标（9人局）
   - 狼人选择袭击目标
   - 女巫查看死讯并选择是否用药
   - 预言家选择查验目标
6. 系统结算，公布 3 条日志
7. 公布死讯
8. 若猎人被刀死 → 猎人开枪环节
9. 警长竞选投票
10. 标记阶段（身份声明 + 评价标记 + 警长归票）
11. 标记公开
12. 放逐投票
13. 公布放逐结果（公开身份）
14. 若猎人被放逐 → 猎人开枪环节
15. 若警长被放逐 → 警徽处置环节
16. 检查胜负

=== 第二天及之后（循环）===

17. 夜晚行动
18. 系统结算，公布日志
19. 公布死讯
20. 若猎人被刀死 → 猎人开枪环节
21. 若警长夜晚死亡 → 警徽处置环节
22. 标记阶段
23. 标记公开
24. 放逐投票
25. 公布放逐结果
26. 若猎人被放逐 → 猎人开枪环节
27. 若警长被放逐 → 警徽处置环节
28. 检查胜负
29. 未结束则回到 17

=== 游戏结束 ===

30. 公布所有玩家身份
31. 显示胜利阵营
32. 展示本局所有标记和投票记录
33. 返回房间大厅
```
