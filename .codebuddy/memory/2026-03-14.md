# 2026-03-14

## 修复：重连时实时操作状态丢失导致游戏卡死

### 问题
刷新浏览器后，虽然有重连逻辑，但当前阶段的实时操作状态（标记轮次、投票、夜晚操作等）没有恢复，导致：
- 标记阶段刷新 → 显示"等待标记阶段开始"
- 投票阶段刷新 → 显示"等待投票阶段"
- 夜晚阶段刷新 → 显示"等待其他玩家操作"
- 如果正好轮到该玩家操作，游戏彻底卡死

### 修复方案
在 GameManager 中添加 `resendCurrentPhaseState(userId)` 方法，重连后根据当前 phase 重新触发对应的回调事件，复用现有事件机制。

### 修改文件
1. `server/game/GameManager.ts` — 新增 `resendCurrentPhaseState()` 和 `getCollectedVotes()` 方法
2. `server/socket/handlers.ts` — 重连后调用 `resendCurrentPhaseState()`，新增 `rebuildInvestigations()` 辅助函数
3. `shared/types/socket.ts` — `ClientGameState` 添加 `investigations` 可选字段
4. `client/src/stores/gameStore.ts` — `setFromReconnect` 补充恢复 investigations 和重置实时状态字段

## 移动端自适应布局重构

### 问题
整个前端没有做任何响应式适配。GameView 使用 `h-screen` + `overflow-hidden` 固定布局，操作面板 `flex-shrink-0` 不能缩小。在手机上，PhaseHeader + PlayerRing + 操作面板的总高度超过屏幕，按钮被挤到屏幕外面，无法操作。

### 修复方案（全面响应式重构）

**核心布局改变**：
- GameView 从 `h-screen + overflow-hidden` 改为 `min-h-screen + overflow-y-auto`（整体可滚动）
- PhaseHeader 改为 `sticky top-0`（固定在顶部）
- PlayerRing 改为响应式半径（基于容器宽度动态计算 + ResizeObserver）
- 所有操作面板的目标选择网格从固定 3 列改为 `grid-cols-2 sm:grid-cols-3`

**修改的文件**：
1. `client/index.html` — viewport 禁止缩放，添加 viewport-fit=cover
2. `client/src/index.css` — 全局移动端 CSS（禁止 tap 高亮、input 16px 防缩放、dvh 支持、安全区域）
3. `client/src/game/GameView.tsx` — 核心布局从固定视口改为可滚动
4. `client/src/game/PlayerRing.tsx` — 响应式半径 + ResizeObserver + 百分比定位
5. `client/src/game/PhaseHeader.tsx` — 紧凑化 + 小屏隐藏次要信息
6. `client/src/game/MarkingPanel.tsx` — 全面紧凑化
7. `client/src/game/NightActionPanel.tsx` — 紧凑化 + 2列网格
8. `client/src/game/VotingPanel.tsx` — 紧凑化 + 2列网格
9. `client/src/game/TriggerPanel.tsx` — 紧凑化 + 2列网格
10. `client/src/game/InfoPanel.tsx` — 固定最大高度 + 更紧凑 tab
11. `client/src/game/GameOverPanel.tsx` — 全面紧凑化
12. `client/src/game/ReplayLog.tsx` — 身份总览1列 + 间距紧凑
